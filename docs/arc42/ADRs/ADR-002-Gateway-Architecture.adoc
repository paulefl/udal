= ADR-002: Structured Monolith Gateway Architecture
:date: 2026-04-28
:status: Accepted

== Context

The gateway must run as a single binary on edge hardware but also scale horizontally in Kubernetes.
Options: structured monolith (one process, modular packages) vs. plugin-monolith (gateway + adapter processes communicating via IPC).

== Decision

**Structured Monolith** for v1: all components in one binary, separated by Go packages with explicit interfaces.
Adapters communicate via the internal `Transport` interface, not IPC.

Weighted score: Monolith **+8**, Plugin **-1**.

== Consequences

* All adapters share the gateway process — a panic in one adapter can affect all
** Mitigation: `recover()` in adapter goroutines, adapter-level circuit breaker
* Future decomposition possible: `Transport` interface is the seam for extraction
* Single binary simplifies edge deployment and Docker image size
